# Cortexa 🧠

**An AI-powered study companion.** Upload your course materials, then chat with them, generate quizzes, summaries, practice sessions and slide decks — all backed by a local Retrieval-Augmented Generation (RAG) pipeline with built-in answer-quality evaluation.


---

## ✨ Features

- **📄 Document ingestion** — Upload PDFs (and other formats) per study session. Documents are parsed with high-resolution layout detection, images are filtered, OCR'd and described with so figures become searchable text.
- **💬 Grounded Q&A chat** — Ask questions about your documents and get streamed answers (SSE) generated by a local **Llama 3.1 8B Instruct** model, with automatic source-page citations. Questions that don't match the documents are politely refused instead of hallucinated.
- **🧩 Concept map** — Chunks are clustered with **HDBSCAN** and the LLM extracts named concepts (definition, keywords, importance) per cluster, which are indexed and used to enrich retrieval.
- **📝 Studio tools**
  - **QCM generator** — multiple-choice quizzes with configurable question count and difficulty.
  - **Practice mode** — the model asks you a question, then grades your answer against the source material.
  - **Summaries** — topic-focused summaries in the language of your choice.
  - **Presentations** — auto-generated slide decks, including relevant document images.
- **📊 Answer evaluation** — Every chat answer is scored in the background for **faithfulness** using an NLI cross-encoder (sentence-level entailment against the retrieved context), plus average rerank scores — all visible in an evaluation dashboard. Offline benchmarking with **RAGAS** is also included.
- **👤 Accounts & personalization** — JWT authentication, per-user sessions (notebooks), profile management, and settings for theme, font size and response length.

## 🏗️ How it works

```
                          ┌──────────────────────── INGESTION ────────────────────────┐
 PDF upload ──▶ unstructured (hi_res) ──▶ image filtering + OCR + Gemini descriptions
                    │
                    ▼
        token-aware chunking (Llama tokenizer)
                    │
                    ├──▶ BGE-M3 embeddings ──▶ FAISS index (per document)
                    │
                    └──▶ HDBSCAN clustering ──▶ LLM concept extraction ──▶ FAISS index (per cluster)

                          ┌──────────────────────── QUERY ────────────────────────────┐
 question ──▶ retrieve top chunks + top concepts ──▶ BGE-reranker-v2-m3 (cross-encoder)
                    │
                    ▼
     relevance gate (refuse if below threshold) ──▶ context builder ──▶ Llama 3.1 8B (4-bit, streamed)
                    │
                    ▼
     background NLI faithfulness scoring ──▶ evaluation dashboard
```

**Models used**

| Role | Model |
|---|---|
| Generation | `meta-llama/Meta-Llama-3.1-8B-Instruct` (4-bit quantized via bitsandbytes, CUDA) |
| Embeddings | `BAAI/bge-m3` |
| Reranking | `BAAI/bge-reranker-v2-m3` |
| Faithfulness (NLI) | `cross-encoder/nli-deberta-v3-small` |
| Image description | Google Gemini |

## 🚀 Getting started


### 1. Configure environment

Create a `.env` file at the project root:

*```env
DATABASE_URL=postgresql-url
JWT_SECRET_KEY=change-me
GEMINI_API_KEY=your-gemini-api-key
HF_API_TOKEN=your-huggingface-token
```*

### 2. Backend

```bash
cd backend
python -m venv .venv
# Windows: .venv\Scripts\activate   |   Linux/macOS: source .venv/bin/activate
pip install -r requirements.txt
python app.py          # starts on http://localhost:5000
```

> ⚠️ On first launch the models (~10 GB) are downloaded from Hugging Face and loaded into memory — this takes a few minutes.

### 3. Frontend

```bash
cd frontend
npm install
npm run dev          
```

### 🐳 Run with Docker instead

```bash
docker compose up --build
```

This starts the frontend on **http://localhost:3000**, the API on **http://localhost:5000**, and PostgreSQL on port 5432. The backend container requires the NVIDIA Container Toolkit for GPU access.

> 💡 The compose file mounts a local Hugging Face cache into the backend container to avoid re-downloading models — adjust the volume path in `docker-compose.yml` to match your machine.

## 🔌 API overview

| Endpoint | Method | Description |
|---|---|---|
| `/auth/signup` · `/auth/login` | POST | Create an account / get a JWT |
| `/auth/sessions/<id>` | GET | List a user's study sessions |
| `/api/upload` | POST | Upload a document to a session |
| `/api/processing` | POST | Parse, chunk, embed, index and cluster a document |
| `/api/chat` | POST | Ask a question — streamed answer (SSE) with sources |
| `/api/studio/qcm` | POST | Generate a multiple-choice quiz |
| `/api/studio/practice/question` · `/evaluate` | POST | Practice mode: get a question / grade your answer |
| `/api/studio/summary` | POST | Generate a summary |
| `/api/studio/presentation` | POST | Generate a slide deck |
| `/api/evaluations/dashboard` | GET | Answer-quality metrics |
| `/profile` · `/settings` | GET/PUT | Profile & preferences |

All routes except signup/login require a `Bearer` JWT.

## 📈 Evaluation

Cortexa doesn't just answer — it checks itself:

- **Online:** after each chat answer, a background thread splits the answer into sentences and verifies each one is entailed by the retrieved context (NLI cross-encoder). The faithfulness score, entailment details and rerank scores are stored per generation.
- **Offline:** `backend/evaluation/` contains a test set and RAGAS benchmark results for the full pipeline.
